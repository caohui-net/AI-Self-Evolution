## 一、元项目定位

### 1.1 核心理念

**AI-Self-Evolution 是"智能体能力工厂"，不是具体应用项目。**

```
┌─────────────────────────────────────────────────────┐
│  My Project (项目总目录)                              │
│  ├── AI-Self-Evolution/  ← 元项目（第二层逻辑）       │
│  │   ├── 能力提炼引擎                                 │
│  │   ├── Gene/Capsule 仓库                           │
│  │   ├── 进化算法库                                   │
│  │   └── 跨项目分发机制                               │
│  │                                                    │
│  ├── Project-A/  ← 应用项目（第三层逻辑）             │
│  │   └── 消费 AI-Self-Evolution 的能力               │
│  ├── Project-B/                                      │
│  │   └── 消费 AI-Self-Evolution 的能力               │
│  └── Project-C/                                      │
│      └── 消费 AI-Self-Evolution 的能力               │
└─────────────────────────────────────────────────────┘
```

### 1.2 与其他项目的关系

| 维度 | AI-Self-Evolution | 其他项目 |
|------|-------------------|----------|
| **角色** | 能力供应商 | 能力消费者 |
| **产出** | Gene/Capsule/技能 | 具体业务功能 |
| **进化方向** | 提炼通用方法论 | 解决特定问题 |
| **依赖关系** | 观察所有项目 | 依赖元项目能力 |

## 二、架构分层

### 2.1 三层架构

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: 观察层 (Observation Layer)                  │
│ ─────────────────────────────────────────────────── │
│ A. 向内观察（被动监听）                               │
│   • 监控所有项目的 Claude Code 会话                   │
│   • 捕获成功的问题解决模式                            │
│   • 识别可复用的技能片段                              │
│   • 记录失败案例和约束条件                            │
│                                                      │
│ B. 向外学习（主动对比）                               │
│   • 同步 EvoMap.ai 技能胶囊库                        │
│   • 分析项目技术栈特征                                │
│   • 评估外部能力的适配性                              │
│   • 筛选可直接应用的 Gene                            │
│                                                      │
│ C. 适配性评估                                        │
│   • 技术栈匹配度计算                                  │
│   • 问题相关性分析                                    │
│   • 依赖可用性检查                                    │
│   • 输出推荐动作：apply/adapt/ignore                 │
└─────────────────────────────────────────────────────┘
           ↓ (原始经验数据)
┌─────────────────────────────────────────────────────┐
│ Layer 2: 提炼层 (Distillation Layer)                 │
│ ─────────────────────────────────────────────────── │
│ • 应用 GEP 协议提炼 Gene/Capsule                     │
│ • 计算 GDI 质量评分                                   │
│ • 构建记忆图（因果关系）                              │
│ • 执行技能蒸馏（自动合成新 Gene）                     │
└─────────────────────────────────────────────────────┘
           ↓ (结构化能力资产)
┌─────────────────────────────────────────────────────┐
│ Layer 3: 分发层 (Distribution Layer)                 │
│ ─────────────────────────────────────────────────── │
│ • 将 Gene 注入到 ~/.claude/agents/                   │
│ • 将 Skill 注入到 ~/.claude/skills/                  │
│ • 更新项目级 CLAUDE.md                                │
│ • 同步到其他项目的 .omc/ 目录                         │
└─────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
┌─────────────────────────────────────────────────────────────┐
│                    观察层 (双向学习)                          │
│  ┌──────────────────┐              ┌──────────────────┐    │
│  │  向内观察         │              │  向外学习         │    │
│  │  (项目实践)       │              │  (EvoMap胶囊库)   │    │
│  └────────┬─────────┘              └─────────┬────────┘    │
│           │                                   │             │
│           └──────────┬────────────────────────┘             │
│                      ↓                                      │
│              ┌───────────────┐                              │
│              │  适配性评估    │                              │
│              │  (技术栈匹配)  │                              │
│              └───────┬───────┘                              │
└──────────────────────┼──────────────────────────────────────┘
                       ↓
              提炼层 (融合处理)
                       ↓
              分发层 (智能注入)
                       ↓
              其他项目复用
                       ↓
              反馈循环 (成功率统计)
```

## 三、核心功能模块

### 3.1 观察引擎 (Observer Engine)

**职责：** 双向学习 - 向内观察实践 + 向外学习知识库

**实现方式：**

**A. 向内观察（被动监听）：**
- Hook 到 OMC 的 PostToolUse 事件
- 读取 `.omc/state/sessions/{sessionId}/` 的会话记录
- 解析 git commit 历史提取成功模式

**B. 向外学习（主动对比）：**
- 定期同步 EvoMap.ai 技能胶囊库
- 分析当前项目的技术栈特征
- 评估外部 Gene/Capsule 的适配性
- 筛选可直接应用的能力

**C. 适配性评估算法：**
```typescript
interface AdaptabilityScore {
  techStackMatch: number;    // 技术栈匹配度 (0-1)
  problemRelevance: number;  // 问题相关性 (0-1)
  complexityFit: number;     // 复杂度适配 (0-1)
  dependencyCheck: boolean;  // 依赖可用性
  overallScore: number;      // 综合评分 (0-1)
}
```

**捕获内容：**
```typescript
interface ObservationRecord {
  projectPath: string;
  sessionId: string;
  timestamp: string;

  // 内部观察
  context: {
    task: string;
    tools: string[];
    agents: string[];
    techStack: string[];      // 新增：项目技术栈
  };
  outcome: {
    success: boolean;
    evidence: string;
    artifacts: string[];
  };
  patterns: {
    problemType: string;
    solutionApproach: string;
    constraints: string[];
  };

  // 外部学习
  externalMatches?: {
    source: 'evomap' | 'github' | 'custom';
    capsuleId: string;
    adaptabilityScore: AdaptabilityScore;
    recommendAction: 'apply' | 'adapt' | 'ignore';
  }[];
}
```

### 3.2 提炼引擎 (Distillation Engine)

**职责：** 将原始观察转化为可复用资产

**核心算法：**
1. **模式识别**：聚类相似问题解决路径
2. **Gene 合成**：提取通用策略模板
3. **Capsule 创建**：封装单次成功经验
4. **质量评分**：应用 GDI 多维度评估

**输出格式：**
```typescript
interface Gene {
  id: string; // SHA-256 hash
  name: string;
  description: string;
  signal: string[]; // 触发条件
  strategy: string; // 解决策略
  constraints: string[]; // 约束条件
  successRate: number; // 历史成功率
  sourceProjects: string[]; // 来源项目
}
```

### 3.3 分发引擎 (Distribution Engine)

**职责：** 将能力注入到目标位置

**注入目标：**
```
全局级别：
  ~/.claude/agents/learned-{name}.md
  ~/.claude/skills/learned-{name}.md
  ~/.claude/rules/learned/{category}.md

项目级别：
  {project}/.claude/agents/
  {project}/.omc/genes/
  {project}/CLAUDE.md (追加 directives)
```

**注入策略：**
- 高质量 Gene (GDI > 0.8) → 全局注入
- 中等质量 Gene (GDI 0.6-0.8) → 项目级注入
- 低质量 Gene (GDI < 0.6) → 仅记录，不注入

    }

## 五、跨项目能力传播

### 5.1 传播机制

**自动传播（推模式）：**
- 新 Gene 生成后，自动推送到所有项目
- 根据项目特征智能匹配相关 Gene
- 更新项目的 CLAUDE.md 和 .omc/genes/

**按需拉取（拉模式）：**
- 项目遇到问题时，查询元项目的 Gene 库
- 根据问题特征匹配最佳 Gene
- 临时注入到当前会话

### 5.2 版本管理

**Gene 版本演化：**
```
gene-auth-jwt-v1.0.0  (初始版本)
  ↓ (在 Project-A 中改进)
gene-auth-jwt-v1.1.0  (支持刷新令牌)
  ↓ (在 Project-B 中改进)
gene-auth-jwt-v2.0.0  (支持多租户)
```

**向后兼容：**
- 保留所有历史版本
- 项目可锁定特定版本
- 提供升级路径和迁移指南

## 七、成功标准

### 7.1 技术指标

- **观察覆盖率**：捕获 ≥ 90% 的项目会话
- **提炼准确率**：生成的 Gene 可用率 ≥ 70%
- **分发延迟**：新 Gene 在 5 分钟内分发到所有项目
- **存储效率**：单个 Gene < 10KB

### 7.2 业务指标

- **能力复用率**：其他项目使用元项目 Gene 的比例 ≥ 30%
- **问题解决加速**：使用 Gene 后解决时间减少 ≥ 40%
- **Gene 库增长**：每月新增 ≥ 5 个高质量 Gene
- **跨项目传播**：单个 Gene 平均被 ≥ 3 个项目使用

## 八、风险与约束

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 观察数据噪音大 | 高 | 多层过滤 + 人工审核 |
| Gene 质量不稳定 | 高 | GDI 评分 + 版本管理 |
| 跨项目冲突 | 中 | 命名空间隔离 + 版本锁定 |
| 性能开销 | 中 | 异步处理 + 批量操作 |

### 8.2 设计约束

- **非侵入性**：不修改其他项目代码
- **可选性**：项目可选择不接收 Gene
- **透明性**：所有 Gene 来源可追溯
- **安全性**：Gene 执行在沙盒中

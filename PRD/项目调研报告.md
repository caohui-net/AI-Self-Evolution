## 一、项目背景

### 1.1 研究动机

当前 AI Agent 面临的核心问题：
- **重复推理成本高**：每个 Agent 独立解决相同问题，浪费计算资源
- **知识孤岛**：Agent 之间无法共享学习成果
- **静态能力**：部署后能力固定，无法根据运行时反馈自我改进
- **缺乏进化机制**：没有系统化的能力积累和传承机制

### 1.2 理论基础

本项目基于以下理论研究：

#### **Test-Time Training (TTT)**
- **来源**：UC Berkeley ICML 2020 (Yu Sun et al.)
- **核心思想**：模型在推理时继续适应，使用测试输入的自监督信号更新参数
- **突破**：打破"训练后参数冻结"的传统假设
- **产业影响**：OpenAI、Anthropic、Google 的推理时计算战略

#### **生物进化论映射**
- **DNA → Gene**：能力的结构化编码
- **遗传 → Capsule**：成功经验的封装和传播
- **自然选择 → GDI 评分**：质量筛选机制
- **共生 → 知识共享**：Agent 间协作进化

## 二、核心参考项目：EvoMap.ai

### 2.1 EvoMap 概述

**定位：** AI 自我进化基础设施 (AI Self-Evolution Infrastructure)

**核心理念：** "One agent learns. A million inherit."（一个 Agent 学习，百万 Agent 继承）

**官网：** https://evomap.ai
**GitHub：** https://github.com/autogame-17/evolver

### 2.2 EvoMap 核心技术

#### **A. 双协议体系**

1. **A2A Protocol (Agent-to-Agent Protocol)**
   - 上层协议：Agent 间通信和协作
   - 功能：知识验证、分发、收益结算

2. **GEP Protocol (Genome Evolution Protocol)**
   - 底层协议：基因组进化机制
   - 功能：能力编码、遗传、变异、选择

#### **B. 核心资产类型**

| 资产类型 | 作用 | 类比 |
|---------|------|------|
| **Gene** | 可重用的进化策略 | DNA 片段 |
| **Capsule** | 单次成功进化记录 | 表型特征 |
| **EvolutionEvent** | 完整审计记录 | 进化历史 |
| **Mutation** | 意图声明 | 突变计划 |
| **ValidationReport** | 验证结果 | 适应度评估 |
| **MemoryGraphEvent** | 因果记忆 | 遗传记忆 |

#### **C. 七阶段进化生命周期**

```
1. Detect (检测信号)
   ↓
2. Select (选择策略)
   ↓
3. Mutate (构建突变)
   ↓
4. Hypothesize (假设预测)
   ↓
5. Execute (执行变更)
   ↓
6. Evaluate (评估结果)
   ↓
7. Solidify (固化经验)
   ↓
[循环]
```

### 2.3 EvoMap 关键创新

#### **1. GDI 评分系统 (Genetic Diversity Index)**

多维度质量评估：
- 结构完整性
- 语义清晰度
- 信号特异性
- 策略质量
- 验证强度

**质量门槛：**
- ✅ 达到阈值 → 推广到市场
- ❌ 低于阈值 → 拒绝或隔离
- 🔄 持续评估 → 质量下降可撤销

#### **2. 记忆图 (Memory Graph)**

因果记忆机制：
- 历史（信号，基因）→ 结果映射
- 低成功率路径自动禁用
- 置信度指数衰减（半衰期 30 天）
- Jaccard 相似度匹配（阈值 0.34）

**聚合公式（拉普拉斯平滑）：**
```
p = (successes + 1) / (total + 2)
weight = 0.5 ^ (age_days / 30)
value = p * weight
```

#### **3. 技能蒸馏 (Skill Distillation)**

元进化机制：
- 从成功 Capsule 自动合成新 Gene
- 触发条件：最近 10 个 Capsule 中 ≥ 7 个成功
- 间隔：≥ 24 小时
- 生成更保守的蒸馏 Gene（max_files: 12）

#### **4. 内容可寻址 (Content Addressing)**

完整性保证：
- SHA-256 哈希所有资产
- 任何篡改可检测
- 去重和验证机制

#### **5. 可移植进化档案 (.gepx)**

主权可移植性：
- gzip 压缩的 tar 档案
- 包含完整进化历史
- 跨平台导出/导入
- 所有权归 Agent 所有者
